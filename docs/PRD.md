* 문서명: 부산 농구 체육관 대관 및 픽업게임 매칭 플랫폼 PRD
* 버전: v0.1 (MVP)
* 작성일: 2026-02-23
* 대상 지역: 부산 (MVP 기준)
* 타겟 사용자:

  * 체육관 사장
  * 농구 동아리/픽업게임 운영자
  * 일반 농구 참가자(게스트)

---

# 1) 기술 환경

## 1.1 플랫폼 범위

* Web 기반 서비스 (모바일 반응형)
* 추후 App 확장 가능 (미정)

## 1.2 사용자 유형

* Admin (운영자)
* 체육관 사장
* 게임 운영자(동아리장)
* 일반 사용자(참가자)

## 1.3 주요 기능 모듈

* 체육관 시간표 관리
* 실시간 예약 가능 여부 표시
* 인원 기반 가격 설정 (예: 5대5 / 소수 인원 할인)
* 픽업게임 모집 게시
* 온라인 결제
* 연락처 직접 노출 제거 (플랫폼 내 신청/결제)

## 1.4 외부 연동 (가정)

* PG사 결제 모듈 (예: 토스페이먼츠 등) — 가정
* 문자 알림 또는 푸시 알림 — 미정

---

# 2) 핵심 차별성

## 2.1 기존 문제

* 인스타그램 시간표 이미지 업로드 방식
* 개인 연락처 문자로 예약/참가
* 계좌이체 기반 수동 결제
* 커뮤니티 게시글 중심 모집 (네이버 카페 등)

## 2.2 차별 포인트

1. 체육관 시간표 실시간 확인

   * 예약 가능 / 정기대관 / 마감 표시
2. 인원 기반 가격 정책 반영

   * 10명 기준
   * 소수 인원 할인 (사장 설정)
3. 연락처 비공개 구조

   * 신청 → 결제 → 확정 흐름
4. 체육관 대관 + 픽업게임 모집 통합

   * 별도 커뮤니티 이동 없음

---

# 3) 사용자 문제 정의

## 3.1 체육관 사장

* 문제 1: 예약 문의가 DM/문자로 분산됨
* 문제 2: 시간표 관리가 수동 이미지 기반
* 문제 3: 인원에 따라 가격 조정 시 반복 설명 필요
* 문제 4: 입금 확인 수동 처리

## 3.2 동아리/픽업 운영자

* 문제 1: 게스트 모집을 카페에 의존
* 문제 2: 개인 연락처 공개 필요
* 문제 3: 인원 현황 관리 어려움 (0/18 수동 표기)
* 문제 4: 입금 확인 및 참가 확정 수동 처리

## 3.3 일반 참가자

* 문제 1: 신뢰 낮은 사이트 이용 불안
* 문제 2: 문자 → 계좌이체 → 확인 대기
* 문제 3: 현재 모집 인원 실시간 확인 어려움
* 문제 4: 취소/환불 기준 불명확

---

# 4) 사용자 흐름

## 4.1 체육관 대관 흐름

1. 사장 회원가입
2. 체육관 정보 등록

   * 위치
   * 코트 수
   * 시간대별 기본 가격
3. 시간표 생성

   * 정기대관 설정
   * 예약 가능 설정
4. 인원별 가격 정책 입력 (예: 10명 미만 시 할인)
5. 사용자 예약 신청
6. 온라인 결제 완료
7. 예약 확정 알림

---

## 4.2 픽업게임 모집 흐름

1. 운영자 회원가입
2. 게임 생성

   * 날짜/시간
   * 장소 (체육관 선택)
   * 최대 인원 (예: 18명)
   * 참가비
3. 모집 상태 표시

   * 모집중 / 마감 / 완료
   * 현재 인원 (예: 1/18)
4. 참가자 신청
5. 결제 완료
6. 자동 인원 카운트 증가
7. 마감 시 자동 모집 종료

---

## 4.3 일반 참가자 흐름

1. 지역 선택 (MVP: 부산)
2. 대관 또는 픽업게임 리스트 조회
3. 시간/가격/인원 확인
4. 신청 버튼 클릭
5. 결제
6. 참가 확정 화면 확인

---

# 5) 결제 구조

## 5.1 체육관 대관

* 시간 단위 결제
* 인원 기반 가격 정책 반영
* 결제 완료 시 예약 확정

## 5.2 픽업게임

* 1인 단가 결제
* 실시간 인원 차감
* 마감 시 자동 신청 차단

## 5.3 정산 구조 (가정)

* 플랫폼 수수료 모델 — 미정
* 체육관/운영자 정산 주기 — 미정

---

# 6) MVP 성공 기준

## 6.1 정량 지표

* 부산 체육관 5곳 이상 입점
* 월 100건 이상 예약/참가 결제
* 문자/계좌이체 방식 대체율 30% 이상 (가정)

## 6.2 정성 지표

* 사장:

  * 시간표 관리가 간편하다고 응답
* 운영자:

  * 모집 관리가 편하다고 응답
* 참가자:

  * 신뢰 가능한 구조라고 응답
