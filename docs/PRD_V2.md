2) 핵심 차별성
- 시간표 기반 예약 구조
  - 체육관의 월간 시간표를 기준으로 예약 가능 여부를 바로 확인 가능
- 운영자 직접 관리 구조
  - 운영자가 시간표, 가격, 할인 정책, 결제 수단을 직접 설정 가능
- 플랫폼 비중개 결제 구조
  - 실제 돈은 플랫폼이 아닌 체육관 운영자에게 직접 송금됨
- 팀 단위 대관 중심 서비스
  - 팀 대표자가 빠르게 체육관을 찾고 예약하는 흐름에 집중
- 실내 체육관 + 야외 농구장 통합 탐색
  - 지도와 목록에서 두 장소 유형을 함께 탐색 가능
- 야외 농구장 리뷰 정보 제공
  - 별점, 시설 상태, 사진 기반으로 현장 상태 확인 가능
- 기존 오프라인 조율 대체
  - 흩어진 인스타, 문자, 개인 연락 중심 예약 과정을 서비스 내 구조로 정리

3) 사용자 문제 정의
- 공통 문제
  - 농구 장소 정보가 분산되어 있음
  - 장소별 상태와 정보 수준이 제각각임
  - 지도와 실제 이용 정보가 연결되어 있지 않음

- 팀 단위 사용자
  - 체육관을 빌리려 해도 빈 시간을 한눈에 보기 어려움
  - 예약 가능 시간, 수업 시간, 정기대관 시간을 구분하기 어려움
  - 가격과 할인 조건을 사전에 알기 어려움
  - 사장님과 개별 연락 후 송금하는 과정이 번거로움

- 체육관 운영자
  - 시간표 관리와 예약 관리가 분리되어 있음
  - 가능한 시간, 수업 시간, 정기대관 시간을 직접 설명해야 함
  - 시간당 가격과 인원 할인 정책을 일관되게 노출하기 어려움
  - 예약자 송금 여부를 수동으로 확인해야 함

- 개인 사용자
  - 야외 농구장의 바닥 상태, 조명, 청결도, 혼잡도를 미리 알기 어려움
  - 실제 방문자 후기와 사진이 부족함

- 관리자
  - 부산 지역 장소 데이터 품질을 유지해야 함
  - 부적절한 리뷰/사진을 관리해야 함

4) 사용자 흐름
- 팀 단위 사용자
  1. 홈에서 장소를 탐색한다.
  2. 지도 또는 목록에서 실내 체육관과 야외 농구장을 확인한다.
  3. 필터를 적용한다.
     - 구/군
     - 장소 유형
     - 예약 가능 여부
  4. 체육관 상세 페이지에 진입한다.
  5. 월간 시간표를 확인한다.
  6. 예약 가능한 시간 슬롯을 선택한다.
  7. 인원을 입력한다.
  8. 할인 적용 여부와 최종 금액을 확인한다.
  9. 예약을 신청한다.
  10. 운영자가 등록한 결제 수단을 확인한다.
      - 카카오페이 송금 링크
      - 은행 계좌번호
      - 예금주명
  11. 운영자에게 직접 송금한다.
  12. 입금자명을 입력하고 송금 완료 처리한다.
  13. 운영자가 입금을 확인한다.
  14. 예약이 확정된다.

- 체육관 운영자
  1. 체육관 정보를 등록하거나 수정한다.
  2. 월간 시간표를 등록하거나 수정한다.
  3. 시간 슬롯 상태를 설정한다.
     - 예약 가능
     - 수업
     - 정기대관
     - 마감
  4. 반복 일정과 예외 일정을 관리한다.
  5. 시간당 가격을 설정한다.
  6. 인원 할인 정책을 설정한다.
     - N명 이하 조건
     - 정률 할인
     - 정액 할인
     - 당일 예약 적용 여부
  7. 결제 수단을 등록한다.
     - 카카오페이 송금 링크
     - 은행 계좌번호
     - 예금주명
  8. 예약 신청 건을 확인한다.
  9. 예약자 정보, 입금자명, 금액, 상태를 확인한다.
  10. 입금 확인 후 예약을 확정 처리한다.

- 개인 사용자
  1. 지도에서 야외 농구장을 탐색한다.
  2. 장소 상세를 확인한다.
  3. 별점, 시설 상태, 사진, 댓글을 확인한다.
  4. 리뷰를 작성한다.
     - 별점
     - 시설 상태 태그
     - 한줄 또는 상세 댓글
     - 사진

- 관리자
  1. 장소 데이터를 등록하거나 수정한다.
  2. 실내 체육관과 야외 농구장 데이터를 관리한다.
  3. 리뷰와 사진을 검수한다.
  4. 부적절한 콘텐츠를 숨김 처리한다.

5) 결제 구조
- 결제 원칙
  - 플랫폼은 결제를 중개하지 않는다.
  - 실제 송금 대상은 체육관 운영자이다.
  - 사용자 간 금전 거래는 다루지 않는다.

- 결제 대상
  - 팀 단위 사용자의 체육관 대관 예약
  - 야외 농구장 리뷰/탐색에는 결제 없음

- 결제 수단
  - 카카오페이 송금 링크
  - 은행 계좌번호
  - 예금주명

- 결제 흐름
  1. 사용자가 예약 신청
  2. 예약 상태를 신청됨 또는 송금 대기로 전환
  3. 사용자에게 운영자 결제 수단 노출
  4. 사용자가 직접 송금
  5. 사용자가 송금 완료 처리
     - 입금자명 입력
     - 송금 완료 버튼 클릭
     - 증빙 이미지 업로드: 선택 사항
  6. 운영자가 입금 확인
  7. 상태 변경
     - 송금 완료
     - 확인 완료
     - 예약 확정

- 예약 상태값
  - 신청됨
  - 송금 대기
  - 송금 완료
  - 확인 완료
  - 예약 확정
  - 취소

- 입금 대기 정책
  - 일정 시간 내 미입금 시 자동 취소 가능
  - 예: 30분
  - 최종 시간 정책은 미정

- 가격 구조
  - 기본 시간당 가격
  - 평일 / 주말 가격: 가정
  - 인원 할인
  - 정률 할인
  - 정액 할인
  - 당일 예약 적용 여부

- 제약 조건
  - 동일 시간 중복 예약 방지
  - 정기대관 시간 예약 불가
  - 수업 시간 예약 불가
  - 마감 시간 예약 불가

6) MVP 성공 기준
- 사용자 측면
  - 부산 내 실내 체육관과 야외 농구장을 탐색할 수 있다.
  - 지도와 목록에서 장소를 구분해 볼 수 있다.
  - 체육관 상세 페이지에서 월간 시간표를 확인할 수 있다.
  - 예약 가능한 시간 슬롯만 선택할 수 있다.
  - 인원 입력 시 할인 적용 여부와 최종 금액을 확인할 수 있다.
  - 운영자 결제 수단을 확인하고 직접 송금 흐름을 완료할 수 있다.

- 운영자 측면
  - 체육관 정보 등록 및 수정이 가능하다.
  - 시간표 등록 및 수정이 가능하다.
  - 시간당 가격 설정이 가능하다.
  - 인원 할인 정책 설정이 가능하다.
  - 결제 수단 등록이 가능하다.
  - 예약 상태와 입금 상태 확인이 가능하다.
  - 입금 확인 후 예약 확정 처리가 가능하다.

- 개인 사용자 측면
  - 야외 농구장의 기본 정보와 리뷰를 확인할 수 있다.
  - 별점, 시설 상태, 사진, 댓글 기반으로 장소 상태를 파악할 수 있다.
  - 리뷰 작성이 가능하다.

- 제품 측면
  - 예약 → 송금 → 확인 → 확정 흐름이 정상 동작한다.
  - 예약 상태값이 정상 전이된다.
  - 중복 예약이 방지된다.
  - 할인 로직이 정상 동작한다.
  - 실내 체육관과 야외 농구장 상세 UI가 구분되어 동작한다.
  - 운영자 입력값이 사용자 화면에 정확히 반영된다.
